MODEL (Logika Data & OOP)
1. User (abstract)

Menyimpan data dasar pengguna dan menjadi parent class untuk semua jenis user.

2. Member

Merepresentasikan anggota perpustakaan yang dapat meminjam dan mengembalikan buku.

3. Librarian

Merepresentasikan petugas perpustakaan yang mengelola data buku dan anggota.

4. Book

Menyimpan informasi buku seperti judul, ISBN, tahun, dan status ketersediaan.

5. Author

Menyimpan data penulis buku.

6. Category

Mengelompokkan buku berdasarkan jenis atau kategori.

7. Loan

Mencatat transaksi peminjaman buku oleh anggota.

8. Fine

Menghitung dan menyimpan denda akibat keterlambatan pengembalian buku.

9. Notification

Interface untuk mengirim notifikasi kepada pengguna (misalnya pop-up).

SERVICE (Logika Bisnis)
10. LibraryService

Mengelola data buku, anggota, dan transaksi peminjaman.

11. AuthService

Menangani proses login dan otorisasi pengguna.

12. FileStorageService

Menyimpan dan memuat data aplikasi ke/dari file CSV.

UI (JavaFX)
13. LoginView

Menampilkan form login dan menerima input pengguna.

14. DashboardView

Menampilkan menu utama dan navigasi fitur aplikasi.

15. BookView

Menampilkan dan mengelola data buku melalui antarmuka GUI.

16. MemberView

Menampilkan dan mengelola data anggota perpustakaan.

17. LoanView

Menangani tampilan peminjaman, pengembalian, denda buku.
